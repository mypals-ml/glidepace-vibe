## Root cause / context
Currently `updateTaskStatus` in `src/hooks/dashboardTasks/useTaskMutations.ts` only updates the Status single-select and progress. No automatic Start Date write happens on transition to a "Done" status.
Existing date infrastructure:
- `src/lib/dateUtils.ts`: `formatToGitHubDate`, `calculate*`
- `src/lib/workCalendar.ts`: `defaultWorkCalendar.formatDate(new Date())` returns accurate *local* YYYY-MM-DD using getFullYear/getMonth/getDate (used for fallbacks and cascades)
- `src/hooks/dashboardTasks/useTaskMutations.ts`: `updateTaskDates` resolves startDate field via `dateSettings.startDateFieldId` / `task.projectFieldIds.startDate` + `findProjectFieldId`, uses `batchUpdateProjectV2ItemFields` + fallback to issue fields
- `updateProjectV2ItemField` / `batch...` in `src/lib/githubService.ts`
- `githubTaskMapper.ts` already defaults missing start to today (but UTC toISOString)
- Status detection regex already present: `/^(done|closed|completed|merged)$/i`

This refine must use local date (not UTC shift) and integrate cleanly without duplicating all resolution logic.
