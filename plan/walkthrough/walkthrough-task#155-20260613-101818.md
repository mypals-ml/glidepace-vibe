# Walkthrough Task #155 - Sync Button Green Dot

## Brief execution walkthrough

Removed permanent green sync badge; sync button now uses icon+timestamp only. Verified focused test, lint, type-check, build, full tests, mock browser header.

## Task

- Issue: #155, `bug: the "sync" button on the main toolbar always has a green dot on it`
- Requirement: find whether the green dot is useful, remove it if useless, or correct it if useful.

## Implementation details

- Inspected the main toolbar sync control in `src/components/Header/SyncStatusIndicator.tsx`.
- Found the status badge was driven by local `isStale` state:
  - `isStale` initialized as `false`.
  - If `lastSyncedTime` was missing or `0`, the staleness check did not update it.
  - That made the non-stale branch render the green dot and ping animation by default.
- Removed the permanent badge overlay from the sync icon.
- Removed the stale timer state from `SyncStatusIndicator`, since the timestamp text already communicates sync freshness.
- Applied the same badge removal to the overflow-menu sync action in `src/components/Header/HeaderOverflowMenu.tsx`.
- Updated `docs/FEATURES.md` to note that sync status is shown without a persistent status-dot badge.
- Added `src/components/Header/SyncStatusIndicator.test.tsx` to cover:
  - the sync button renders without `.bg-emerald-500`, `.bg-emerald-400`, or `.animate-ping` badge classes.
  - clicking the sync button still refreshes the selected project.
