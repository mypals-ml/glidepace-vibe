# Task #160 Walkthrough

## Summary

Enabled desktop Gantt task-bar dragging: horizontal drags shift task start dates through the existing date update cascade, while vertical drags reuse the sidebar reorder/group move planners and persistence mutations.

## Implementation Details

- Added a Gantt drag planner in `src/components/Dashboard/Views/ganttChartUtils.ts`.
- The planner computes a shifted start date from horizontal day deltas.
- The planner resolves vertical drop targets against the rendered dashboard rows and reuses existing group drop, group-path move, and visible reorder plans from `taskOrderUtils`.
- Added pointer drag state to `GanttChart`.
- Task bars now show a live translated preview while being dragged.
- Mouse drags on task bars are separated from existing dependency-link handles via `data-gantt-link-handle`.
- On drop, Gantt bars call:
  - `updateTaskDates` for horizontal start-date changes.
  - `moveTaskToGroupPath` for group or field-group moves.
  - `reorderTask` / `reorderTaskBlock` for normal vertical reorders.
- Preserved existing touch long-press context menu behavior.
- Moved shared Gantt helper logic into a non-component utility module so React fast-refresh lint remains clean.
- Updated `docs/FEATURES.md` to describe Gantt task-bar drag behavior.
