Design notes:

- A `DashboardTasksCore` object threads the shared context (token, selected
  project, date settings + ref, tasksRef, setTasks, sync/toast callbacks)
  through every sub-hook, so all modules mutate the same synchronously-
  tracked task list exactly as before.
- All sub-hooks are called unconditionally in a fixed order, preserving the
  Rules of Hooks; each sub-hook owns only the state it previously owned
  inside the monolith.
- Hook return object keys are byte-for-byte identical to the original, and
  `FetchProjectTasksOptions` is re-exported from the original module path, so
  no consumer changed.
- The React Compiler lint rule (`react-hooks/preserve-manual-memoization`)
  required extracting `selectedProjectId = selectedProject?.id` locally in
  `useTaskGrouping`/`useTaskOrdering` instead of using optional chaining in
  dependency arrays; memoization granularity is unchanged.
