# Walkthrough: Task #160

## Task

Feature: allow drag and drop the task bar in gantt chart view

Reviewer refinement handled:

- When moving a Gantt task bar up and down, the whole row should move visually like task-list row movement.
- In mobile view, the Gantt task-bar context menu should include a "Move task" action. Selecting it arms that task bar for movement; tapping outside exits the move state.

## Implementation

- Updated `src/components/Dashboard/Views/GanttChart.tsx`.
- Added a mobile breakpoint check with `useMediaQuery('(max-width: 767px)')`.
- Added `mobileMoveTaskId` state for touch movement arming.
- Added a mobile-only `Move task` item to the Gantt task-bar context menu.
- Selecting `Move task` marks the task as selected, closes the menu, and sets the bar to a move-ready state.
- Touch pointer dragging is only enabled for the armed mobile task bar; ordinary touch still opens the long-press context menu.
- Tapping outside the armed task bar exits the move state.
- During active drag, vertical translation is applied to the whole Gantt row while horizontal translation remains on the task bar. This keeps the row, selection highlight, and connectors moving together vertically.
- On completed or cancelled mobile drag, the move state is cleared.
- Updated `docs/FEATURES.md` to document row-level Gantt drag feedback and mobile Gantt task-bar movement.
