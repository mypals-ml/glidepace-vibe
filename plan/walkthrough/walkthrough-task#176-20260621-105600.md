## Brief execution walkthrough

Raised header/dropdown z-index above task details overlay; inline panel no longer stacks at z-50. Tests/lint/build passed.

## Task

- GitHub issue: https://github.com/mypals-ml/glidepace-vibe/issues/176
- Title: bug: the drop down list of the main tool bar

## Root cause

Header toolbar dropdowns (project, language, overflow menu) used `z-50`, the same level as the task details overlay panel. When task details were open, the panel painted over the dropdown menus.

## Changes made

### `src/lib/uiLayering.ts`
- Centralized stacking classes: task details backdrop (`z-40`), panel (`z-50`), header (`z-[60]`), header dropdowns (`z-[70]`)

### Header components
- `Header.tsx` — header layer raised to `z-[60]`
- `LanguageSelectorDropdown.tsx` — dropdown menu uses `z-[70]`
- `ProjectSelectorDropdown.tsx` — dropdown menu uses `z-[70]`
- `HeaderOverflowMenu.tsx` — overflow menu uses `z-[70]`

### `TaskDetailsPanel.tsx`
- Overlay mode keeps backdrop/panel at `z-40`/`z-50`
- Inline (desktop) mode uses `z-0` instead of `z-50` since it sits in normal layout flow

### Tests
- `src/lib/uiLayering.test.ts` — asserts header layers stack above task details
