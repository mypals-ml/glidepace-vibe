## Implementation

- Added `resolveFieldGroupedTargetPath` in `src/lib/taskOrderUtils.ts`.
  - It separates visible field-group prefixes such as `Status: Todo` from the real group path such as `group1`.
  - It returns field value changes separately from the group path that should be persisted to the `Group Path` text field.
  - It handles named field-group labels and `No value` labels.
- Threaded the active `Group by Fields` context through `TaskSidebar` drag/drop planning.
  - Group header drops and task-over-task group moves now receive selected group field IDs and field metadata.
  - Drop-target highlighting uses the same normalized path logic as the final drop plan.
- Extended `moveTaskToGroupPath` in `useDashboardTasks`.
  - It accepts optional field value changes from the planner.
  - It optimistically updates local task field values and status/progress when the changed field is the Status field.
  - It persists single-select values using option IDs when available, text field values otherwise, and clears fields for empty values.
  - It persists the normalized real `Group Path`, so display prefixes are not written into the Group Path field.
- Updated `DashboardContext` to expose the extended move function signature.
- Updated `docs/FEATURES.md` to document moving between field groups.
