## What was implemented

### New module `src/lib/fieldGroupHistory.ts`

- `getUsedFieldGroupsStorageKey(projectId)` — per-project localStorage key
  (`used_field_groups_<projectId>`, falling back to `used_field_groups`).
- `loadUsedFieldGroups(storage, key)` — loads, validates, normalizes, dedupes
  (identical = same fields in the same order), and sorts history entries by
  `savedAt` descending; tolerates malformed/absent storage content.
- `recordUsedFieldGroup(storage, key, fieldIds, savedAt)` — prepends the freshly
  applied combination, merges identical entries (keeping the newest timestamp),
  caps the list at `MAX_USED_FIELD_GROUPS` (20), persists it, and returns the
  updated list. Empty/whitespace-only selections are ignored.
- `getRecommendedFieldGroups(fields)` — resolves the preset name combinations
  (`Status`; `Status, Priority`; `Priority, Status`) against the project's actual
  fields case-insensitively; a combination is only offered when every referenced
  field exists.
- `areFieldIdListsIdentical`, `getLocalStorageSafe` helpers.

### `src/components/Dashboard/TaskSidebar.tsx`

- The Group by Fields dialog now renders a three-tab bar (`role="tablist"` with
  proper `aria-selected` / `aria-controls` wiring): **All fields** (the existing
  search + checkbox list), **Used groups**, and **Recommended**.
- Opening the dialog resets to the All fields tab and loads the per-project used
  history from localStorage.
- Used/Recommended tabs list each combination as a clickable row of field-name
  chips joined by chevrons; clicking loads that combination into the current
  draft selection (highlighted with a check when it matches exactly). Empty
  states use dedicated i18n strings.
- Saving a non-empty selection records it into the used-groups history before
  applying the grouping.

### `src/i18n.ts`

  `fieldGroupTabUsed`, `fieldGroupTabRecommended`, `noUsedFieldGroups`,
