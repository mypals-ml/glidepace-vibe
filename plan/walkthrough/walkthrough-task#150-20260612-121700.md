## What was implemented

### `src/components/Dashboard/TaskDetailsPanel.tsx`

- The `{/* Task Actions */}` card (Add task above / Add task below /
  Add Successors / Group buttons, plus the conditional break-links area) was
  removed from its previous position between the Description card and the
  Status section.
- The card is now rendered at the end of the details content, after the
  Comments section and immediately before the bottom spacer, so the section
  order is now: Title, Description, Status, Assignees, Repository, Dates /
  Estimate / Auto-update, Progress, Comments, **Task actions**.
- The previously standalone Delete Task block (`border-t border-red-100`
  wrapper with the danger `Button`) was removed and re-homed inside the Task
  actions card as a final `border-t border-red-100 p-3` footer row, keeping the
  same button behavior (`setIsTaskDeleteConfirmOpen(true)`, disabled/loading
  while `isDeletingTask`) and the same confirmation modal.
- No logic, i18n keys, or handlers were changed — this is purely a re-ordering
  of JSX inside `TaskContent`'s details branch. Both the mobile and desktop
  layouts render `TaskContent`, so both pick up the new order.

### `src/components/Dashboard/TaskDetailsPanel.test.tsx`

Two new tests (existing four still pass):

1. **renders the Task actions section at the end of the view, after the
   comments section** — renders the panel and asserts via
   `compareDocumentPosition` that the "Task actions" label follows the
   Description, Status, and Comments section labels in DOM order (scoped to the
   first of the two layout copies).
2. **renders the Delete Task action inside the Task actions section** — locates
   the Task actions card element and asserts the Delete Task button and the
   Add-task-above action both live inside that same card.

Also switched the `@testing-library/react` import to include `within`.

### `docs/FEATURES.md`

- "Task Deletion" and "Task Detail Actions" entries updated to document that
  the Task actions section now sits at the end of the task details view and
  hosts the delete-task danger zone (RULES.md documentation-maintenance rule).
