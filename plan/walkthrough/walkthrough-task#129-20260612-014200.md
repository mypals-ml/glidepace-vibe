# Walkthrough — Task #129: refine: show link lines even when group card is folded

## Brief execution walkthrough

Dependency lines now survive group folding in the Gantt chart. Links that
cross a folded group card's boundary anchor to the card's title bar: a link
from an outside predecessor points to the card, and a link from a hidden task
to an outside successor starts from the card. Links fully inside one folded
card stay hidden, and duplicate lines to the same outside counterpart collapse
into a single line. Verified with 6/6 unit tests (5 new), strict `tsc -b`, and
eslint.

## Background / problem

`dashboardItems` (built by `buildGroupBlocksFromOrderedTasks`) omits every
descendant row of a collapsed group block. `DependencyLines` resolved both
link endpoints exclusively from the visible rows' bounds map, so as soon as a
group card was folded, any link touching a task inside it silently vanished —
even though the task description requires those links to remain visible,
re-anchored to the folded card's title bar.

## What was implemented

### `src/components/Dashboard/Views/DependencyLines.tsx`

- New optional prop `tasks?: Task[]` — the full (filtered) task list including
  tasks hidden inside folded cards. Falls back to the visible tasks in `items`
  when omitted (keeps the existing component contract and tests working).
- New `LineAnchor` type (`x1`, `x2`, `y`, optional `groupBlockId`) shared by
  the existing task `boundsMap` and the new anchors.
- New `foldedCardAnchorMap` memo: for every visible **collapsed** group block
  with dates, computes the card title bar bounds and registers them for each
  of the group's `childTaskIds`. Geometry mirrors the GanttChart card exactly
  (`GROUP_CARD_PAD = 9`, `GROUP_CARD_MIN_SPAN_WIDTH = 120`, y at the row
  center, which is the vertical center of the 40px title bar in the 72px row).
  Nested folded groups are inherently handled: only the outermost collapsed
  block appears in `items`, so each hidden task maps to exactly one card.
- Line building now iterates `tasks ?? items.filter(isDashboardTask)` and
  resolves each endpoint from `boundsMap` first, then `foldedCardAnchorMap`.
  - If both endpoints resolve to the same folded card, the link is internal to
    the group and is skipped (hidden together with its tasks).
  - Lines whose anchor pair (folded card / task) was already drawn are
    deduped, so several hidden tasks linking to one outside counterpart render
    as a single line instead of identical overlapping copies. The break-link
    icon still operates on the underlying represented link.
- Existing behavior (visible-to-visible lines, drag preview, snap-to-target,
  break-link hover icon, dash animation) is unchanged.

### `src/components/Dashboard/Views/GanttChart.tsx`

- Passes `tasks={filteredTasks}` to `DependencyLines` so links involving
  folded-away tasks can still be discovered.

### `src/components/Dashboard/Views/DependencyLines.test.tsx`

Five new tests (plus the pre-existing drag-preview test still passing):
1. Fallback: visible task-to-task line still drawn when no `tasks` prop given.
2. Outside predecessor → hidden task: line ends at the folded card's left
   edge at title-bar height (asserted exact coordinates `231, 108`).
3. Hidden task → outside successor: line starts at the folded card's right
   edge (`609, 36`) and ends at the successor bar start (`600, 108`).
4. Link fully inside one folded card renders no line.
5. Two hidden tasks linking to the same outside successor render exactly one
   line.

### `docs/FEATURES.md`

- Task Dependencies section documents the folded-card link-line behavior
  (required by RULES.md documentation-maintenance rule).
