# Task #37: Refactor the Burndown Chart feature

## Brief execution walkthrough

Implemented the Burndown view as a real Chart Wall 5 style dashboard with forecast metrics, a remaining-effort basin chart, status-days donut, and worker daily load bars. Verified calculation behavior, lint, type-check, production build, and a mock browser render.

## Implementation notes

- Replaced the placeholder Burndown screen with a responsive dashboard in `src/components/Dashboard/Views/BurndownChart.tsx`.
- Added reusable burndown math in `src/lib/burndownChartUtils.ts`.
- Calculates estimate days from task estimate/unit when present, and falls back to inclusive start/target duration.
- Normalizes date-only and timestamp task dates so mock and real project data both render safely.
- Splits work into Done, In progress or review, and Todo buckets using status/progress.
- Builds actual/projected remaining-effort points from the filtered task set.
- Builds top worker daily load bars for the next 10 days from open work.
- Added localized labels for English, Japanese, Simplified Chinese, and Traditional Chinese.
- Updated `docs/FEATURES.md` to document the implemented Burndown dashboard.
