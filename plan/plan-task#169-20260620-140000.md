# Implementation Plan: Task #169 — Gantt dependency link hover

## Task summary

- Issue: #169 — Hover on the start half of a dependency link from task #1 does not turn the line red or show the unlink button; the end half works.
- Repro project: `PVT_kwDODNQhvs4BY49y`

## Root cause

`DependencyLines` renders before task rows, but each gantt row wrapper spans the full timeline width with default `pointer-events: auto`. Those row layers sit above the SVG and intercept hover events along the source row, which covers the first half of outgoing links.

## Implementation

1. Set `pointer-events-none` on task and group row wrappers in `TimelineChart.tsx`.
2. Restore interactivity with `pointer-events-auto` on task bars, group title buttons, and link handles.
3. Add a regression test in `TimelineChart.test.tsx`.
