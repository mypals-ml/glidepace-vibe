# Implementation Plan: Task #164 — Mobile header icon margins

## Task summary

- Issue: #164 — Change the icon of main page at the top left corner in mobile view mode
- Latest refinement: keep the margin between the compact `G` app mark and nearby header elements when the header compresses.

## Root cause

Header overflow compression lowers `--header-gap` (8px → 4px → 2px). The title block and divider previously used that variable, so spacing around the mobile icon shrank on narrow/overflow layouts.

## Implementation

1. Add `--header-divider-gap: 8px` in `src/index.css` (not overridden by `.header-compressed`).
2. Remove compressible `marginRight` from the title wrapper in `Header.tsx`.
3. Apply fixed `marginLeft` and `marginRight` on the divider via `--header-divider-gap`.
4. Keep the divider always visible (remove `hidden sm:block`).
5. Add a regression test in `Header.test.tsx`.
6. Document behavior in `docs/FEATURES.md`.

## Risks / gaps reviewed

- Overflow items still compress via `--header-gap`; only icon/divider spacing stays fixed.
- Divider remains `shrink-0` so it will not collapse.
- No impact on desktop full app name layout.
