# Feature Specifications

## Currently Implemented
*   **Connect to GitHub Projects:** Connects to your GitHub.com projects to visualize tasks.
*   **Authentication & Multiple Accounts:** Provides a starting page to authenticate securely via GitHub OAuth. Users can connect multiple GitHub accounts, store them locally, and seamlessly switch between them.
*   **Project Discovery & Selection:** Instead of manually inputting a Project ID, authenticated users can browse their active projects via a dedicated "Open Project" modal, with selections cached in `localStorage`.
*   **Real-time Sync:** Automatically receives updates from GitHub via webhooks and Supabase broadcast, updating the UI instantly without page refreshes.
*   **Localization (i18n):** Fully translated interface supporting English, Japanese, and Simplified Chinese.
*   **Header About Entry:** The top toolbar now includes an overflow-aware `About` button that opens a lightweight product overview modal with a direct link to the source repository.
*   **Themes:** Supports Light and Dark mode toggling. The default interface is a polished Bright (Light) mode with a glassmorphism aesthetic.
*   **MS Project Style Layout:** The main dashboard uses a resizable split-pane layout: the left panel lists issues (with assignees and status), while the wide right panel displays a horizontal Gantt chart.
*   **Status-Colored Task IDs:** Task IDs in the issue list use the same color family as each task's current status, making status scanning easier without adding another column.
*   **Jump to Chart:** Long-pressing or right-clicking a task in the list opens a context menu with "Jump to Chart", switching to the Gantt chart and centering the selected task in view.
*   **Sync Status & Manual Sync:** Automatically tracks the last synchronization time with GitHub and displays it in the header (e.g., "Synced 5 minutes ago"). Users can hover over the status to reveal a "Sync Now" button for manual synchronization.
*   **Robust Demo Mode:** Full functionality available offline via mock services (`githubMock.ts`) for testing and demonstration purposes. Supports simulated GraphQL queries and mutations, with in-memory project field persistence for edits such as task dependency links until the app is quit.
*   **Task Deletion:** Task details include a danger-zone delete action with confirmation. Confirming removes the task from the GitHub Project and deletes the linked repository issue when one exists.
*   **Task Date Editing:** Task details let users set a fixed real start date or use "Reset to Auto" to clear the persisted GitHub start date. After reset, the start date input displays the auto-calculated date from the current scheduling and dependency rules.
*   **Task Reordering & Positioned Creation:** Tasks keep the default order returned by GitHub Projects and can be reordered from the sidebar using drag-and-drop. The app persists order through GitHub Projects V2's native item position mutation, so large moves only update the moved item position instead of rewriting custom order fields. Filtered task lists can also be reordered relative to visible neighbors. On mobile, long-pressing a task opens the context menu; choosing "Move" reveals the reorder handle for that task. Long-pressing a task on mobile, or opening the desktop row "more" menu, also lets users create a new task directly above or below the selected task. In desktop mode, cancelling or completing positioned task creation now fully closes the right-side create panel so the chart immediately expands back into the freed space.
*   **Task Grouping:** Tasks can be grouped by a GitHub Projects text field named `Group Path`, which stores a JSON string array such as `["Planning","Research"]`. The root group is implicit and displayed as the current project name. Groups are derived from contiguous task-order runs, so two separated blocks with the same group name are treated as separate visible group blocks. Group rows support expand/collapse, rename, ungroup, context-menu actions, and drag reordering; moving a group moves the whole visible group block, including nested child group tasks. Individual tasks can also be dragged onto a group row to move into that group, onto another group row to switch groups, or onto the root project group to clear their group path. Users can also open `Group by Fields` from the task-list title bar, choose one or more GitHub Project fields, reorder the selected fields in the dialog, and apply them as temporary visible grouping prefixes while preserving each task's existing `Group Path` grouping inside those field-value groups. Group start and target dates are calculated from the earliest start and latest target date among tasks in that visible group block. The sidebar renders nested groups with palette-colored circular tree nodes and elbow connectors that align group and task rows while stopping branch rails at the final child joint.
*   **Work Calendar:** Scheduling uses a default work calendar that reports the local browser timezone and treats Monday through Friday as workdays. Dependency successor dates skip non-workdays, while task duration estimates still calculate target dates in calendar days. The Gantt chart shades non-workday columns with a stronger muted background.
*   **Task Dependencies:** Supports finish-to-start successor links between tasks, storing both successor and predecessor text fields so links and unlinks can recalculate affected tasks efficiently. Normal dependency walks auto-correct missing reverse links when one side is discovered, while explicit unlink operations only clear the requested link. In task-list and Gantt task-bar context menus, users can break all related links for the selected task or block boundary, or break only predecessor or successor links; those actions only appear when matching links exist. Dependency-derived dates cascade through display-only temporary start/target dates so GitHub date fields are not auto-modified. Tasks with GitHub start dates are treated as fixed starts and use a per-project ask/auto display update setting; tasks without GitHub start dates always update their temporary display dates. In the Gantt view, dependency lines keep a subtle slow-moving dash animation so links stay readable without feeling visually busy. When dragging to link tasks, the connector line snaps to the target node, changing to a thin green/emerald line with accelerated flow speed, and the target connector node uses a subtle emerald ring and pulse to confirm the drop target before release. On mobile, the link tasks builder replaces the active view's bottom control bar instead of floating above the task list or Gantt chart.
*   **Mobile Back Gesture Handling:** In mobile view, browser back gestures first dismiss the active dialog or task detail overlay. If no dialog is open and the user is viewing a chart, the gesture returns to the task list instead of leaving the app.
*   **Assignee Search Strategy:** 
    *   **Contextual Priority:** The search interface automatically prioritizes "Project Mates" — users already assigned to at least one task in the current project for quick access.
    *   **Smart Scoping:** For projects owned by an Organization, the system automatically scopes the GitHub user search to that organization (`org:LOGIN`) to find relevant teammates quickly.
    *   **Global Discovery:** For personal projects that are **public**, the system performs a global GitHub user search using the GraphQL `search(type: USER)` API. For private personal projects, search is restricted to project mates for security.
    *   **Real-time Debounce:** User search occurs dynamically as you type with a 500ms debounce to optimize API calls while providing a responsive "instant search" feel.

## Roadmap & Upcoming Features
*   **Data Persistence (Custom Fields):** Saves the necessary information back in the project, including creating custom attributes for tasks.
*   **Draft Issue Conversion & Setup:** Automated workflow to convert "Draft Issues" to full Issues to enable advanced metadata and dependencies.
*   **Save Gantt Chart as View:** Ability to save the current Gantt configuration as a view in GitHub for easy sharing and persistence.
